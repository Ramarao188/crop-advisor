# Annadata — AI Crop Advisor

A two-stage machine learning system that recommends the best crop to grow based on soil and weather conditions, then forecasts the expected yield for that crop.

**Live demo flow:** enter soil nutrients (N-P-K, pH) and weather (rainfall, temperature) → get the top 3 recommended crops with confidence scores → pick one → enter field-level detail (region, irrigation, fertilizer use) → get a yield forecast in tons/hectare with a risk assessment.

## Why this project

Most crop-prediction projects stop at a single model. This one chains two models into a single decision-support pipeline, closer to how a real agri-tech product would work: first narrow down *what* to plant, then estimate *how much* you'll get from it.

## Architecture

```
┌─────────────────┐      ┌──────────────────┐      ┌─────────────────┐
│   Frontend       │──────▶│   Flask API       │──────▶│  Two ML Models   │
│  (HTML/CSS/JS)   │ HTTP │  (REST endpoints)  │      │  (XGBoost x2)    │
└─────────────────┘      └──────────────────┘      └─────────────────┘
```

**Stage 1 — Crop Recommendation (Classification)**
- Model: XGBoost Classifier
- Input: N, P, K, soil pH, rainfall, temperature
- Output: Top 3 crops out of 40, with confidence %
- Performance: 93.6% test accuracy, 93.4% weighted F1 (5-fold CV: 93.6% ± 0.1%)
- Dataset: 18,079 records

**Stage 2 — Yield Prediction (Regression)**
- Model: XGBoost Regressor (in a scikit-learn pipeline with scaling + one-hot encoding)
- Input: 19 features — soil type, soil chemistry, weather, region, season, irrigation type, fertilizer/pesticide usage
- Output: Predicted yield in tons/hectare + qualitative risk flag
- Performance: R² = 0.998, MAE = 0.84 tons/hectare (5-fold CV R²: 0.998 ± 0.0001)
- Dataset: 10,000 records, 6 major crops (Rice, Wheat, Potato, Cotton, Maize, Sugarcane)
- Note: the yield model currently covers 6 crops. If a recommended crop from Stage 1 isn't one of these six, the UI clearly flags it rather than producing an unsupported prediction.

## Tech stack

- **ML**: Python, scikit-learn, XGBoost, pandas, joblib
- **Backend**: Flask, Flask-CORS (REST API)
- **Frontend**: vanilla HTML/CSS/JavaScript (no framework — fetch API for HTTP calls)

## Running it locally

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Train the models
```bash
cd notebooks
python3 train_recommendation_model.py
python3 train_yield_model.py
```
This saves model artifacts into `models/`.

### 3. Start the backend
```bash
cd backend
python3 app.py
```
Runs on `http://localhost:5000`.

### 4. Open the frontend
```bash
cd frontend
python3 -m http.server 8080
```
Then open `http://localhost:8080` in your browser.

## API endpoints

| Method | Endpoint              | Description                                  |
|--------|-----------------------|-----------------------------------------------|
| GET    | `/api/health`         | Health check                                   |
| GET    | `/api/metadata`       | Model metrics + dropdown options for the UI    |
| POST   | `/api/recommend`      | Crop recommendation given soil/weather inputs  |
| POST   | `/api/predict-yield`  | Yield forecast given crop + field conditions   |

## Deployment

This project deploys as two separate services:

**Backend (Flask API) → Render**
1. Push this repo to GitHub
2. Go to [render.com](https://render.com) → New → Web Service → connect your GitHub repo
3. Render auto-detects `render.yaml` and configures the build/start commands
4. Deploy — you'll get a URL like `https://annadata-backend.onrender.com`

**Frontend → Vercel**
1. Go to [vercel.com](https://vercel.com) → New Project → import the same GitHub repo
2. Set **Root Directory** to `frontend`
3. Before deploying, update `frontend/app.js` — change `API_BASE` to your live Render URL from above (with `/api` at the end)
4. Deploy — you'll get a URL like `https://annadata.vercel.app`

Note: Render's free tier spins down after 15 minutes of inactivity, so the first request after idle time may take 30-50 seconds to wake up — this is normal and worth mentioning if you're demoing it live.
